# KLO Local Mac App

The native Swift/SwiftUI app for KLO Local. It owns the notch UI, global hotkeys,
permission UX, local sidecar launch, and run state. The agent loop itself lives
in `agent2` and is reached over the local sidecar contract at
`http://127.0.0.1:8787`.

## Quick Start

```bash
brew install xcodegen
cd desktop-mac
xcodegen generate
open KLO.xcodeproj
# Hit вҢҳR in Xcode to build + run, or:
xcodebuild -project KLO.xcodeproj -scheme KLO -configuration Debug -destination "platform=macOS" build
```

Use `вҢҳK` to open KLO from anywhere.

## Hotkeys

- `вҢҳK` вҖ” toggle text-input panel
- `вҢҳвҮ§K` вҖ” toggle voice-input panel
- `Escape` вҖ” collapse expanded panel back to the idle line
- `Return` вҖ” submit query

Hotkeys work from other apps without stealing focus until KLO needs text input.

## Permissions

KLO uses macOS permissions for user-initiated tasks:

- **Accessibility** вҖ” move the cursor, click, type, and press keys.
- **Screen Recording** вҖ” inspect the current screen during a run.
- **Microphone** вҖ” voice mode only.
- **Apple Events** вҖ” interact with scriptable apps when needed.

Run `klo-doctor` from the repo root to verify the runtime permission state.

## Distribution

Public local builds use automatic/ad-hoc signing by default. If you distribute a
fork, use your own Apple Developer identity and notarization credentials.

To notarize:

```bash
export KLO_TEAM_ID=YOUR_TEAM_ID
export KLO_NOTARY_PROFILE=your-notary-profile
export KLO_SIGNING_IDENTITY="Developer ID Application: Your Name (TEAMID)"
./Distribution/notarize.sh
```

## Integration Points

`KLO/Agent/AgentClient.swift` is the seam between Swift UI state and the local
agent sidecar:

- `submitQuery(text)` вҶ’ POST `http://127.0.0.1:8787/runs` + open WebSocket to `ws://127.0.0.1:8787/ws/runs/{id}`. Listen for event types `tool_call`, `tool_result`, `step_progress`, `final_message`, `error`. Drive `KLOState` transitions on `final_message` and `error`.
- `cancelCurrentRun()` вҶ’ POST `/runs/{id}/cancel`.
- `submitConfirm(approved:)` вҶ’ POST `/runs/{id}/confirm`.

UI files should not talk to the network directly; keep sidecar interaction in
`AgentClient`.

## Notch Detection

`KLO/Detection/NotchDetector.swift` reads `NSScreen.safeAreaInsets`, `auxiliaryTopLeftArea`, `auxiliaryTopRightArea`. Recomputes on `NSApplication.didChangeScreenParametersNotification`. The window controller observes via Combine; views observe by holding the detector and reading `geometry`.

For non-notched displays (Mac mini, external monitor, pre-2021 MBP), `geometry.hasNotch == false` and `IdleLineView` falls back to a 280Г—4 pill at top-center.
